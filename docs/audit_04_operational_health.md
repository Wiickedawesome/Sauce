## Section 6 — Daily Stats Completeness

### Evidence

**Active DB:** 0 rows in `daily_stats`.

**Backup 2:** 1 row in `daily_stats`.

| date | loop_runs | signals | vetoed | orders | realized_pnl | starting_nav | ending_nav | paused | updated_at |
|------|-----------|---------|--------|--------|-------------|-------------|-----------|--------|------------|
| 2026-03-12 | 1 | 7 | 0 | 0 | 0.0 | 0.0 | 798.82 | 1 | 2026-03-12 17:43:29 |

**Days with loop activity in backup 2:** 6 days (Mar 8-13).  
**Days with daily_stats rows:** 1 day (Mar 12 only).  
**Missing daily_stats for:** Mar 8, Mar 9, Mar 10, Mar 11, Mar 13 (5 of 6 days).

**Root cause:** The `upsert_daily_stats()` call in `ops.py` runs at the end of each loop. With 427 test loops across 6 days, only 1 stats row was persisted. This suggests:
1. The ops agent only ran to completion in 1 of the 427 loops (the rest likely errored out before reaching the stats upsert), OR
2. The `upsert_daily_stats` function itself failed silently for most runs

Given that only 2 `ops_summary` events exist (both on Mar 12), hypothesis #1 is confirmed — the ops agent only reached full execution on Mar 12 in one loop.

**`upsert_daily_stats` code review:** The caller passes `loop_runs=1` per invocation, and the function correctly accumulates additive fields (including `loop_runs`) across multiple calls for the same date. This design is correct — each loop run increments the counter by 1.

### Findings

| ID | Severity | Finding |
|----|----------|---------|
| F-14 | WARN | **Daily stats table is 83% incomplete.** Only 1 of 6 active days has a `daily_stats` row. The ops agent did not reach full execution on 5 of 6 days (consistent with the high API error rate preventing full loop completion). |
| F-15 | PASS | **`loop_runs` accumulation is correct.** `upsert_daily_stats()` correctly accumulates additive integer fields (loop_runs, signals_generated, signals_vetoed, orders_placed) across multiple calls per day. Passing `loop_runs=1` adds 1 to the daily total — not an overwrite. |
| F-16 | INFO | **Active DB has zero daily stats.** Expected given zero loop runs. Not a bug — just confirms pre-production state. |

### Verdict: **WARN**

Daily stats pipeline exists and is correctly implemented. Incompleteness (83%) is due to the ops agent not reaching full execution on most test days (API errors prevented completion), not a code defect.
